Lesson 16

Python String Formatting and Comments

In previous lessons we learned variables, input/output, operators, conditionals, data structures, and loops. Now we learn strings, string formatting, and comments — tools that make programs more readable and professional.

By the end of this lesson you will be able to

  • Understand the concept of strings in Python with its practical uses
  • Format strings with different string formatting methods in Python
  • Access string characters using indexing and slicing
  • Write comments to improve code readability
01

Basic String Operations

Before learning string formatting, we need to understand one important fact: a string is made of individual characters.

name = "Nimal"

This string contains 5 characters: N i m a l

Python gives each character a position number called an index.

02

String Indexing

The index starts from 0.

CharacterNimal
Index01234

We can access characters using square brackets:

indexing.py 
name = "Nimal"
print(name[0])   # first character
print(name[2])   # third character
Output
N
m
Remember Python starts counting from 0, not 1. So name[0] is the first letter and name[1] is the second.

Negative indexing — Python can also count from the end:

print(name[-1])   # last character
Output
l
03

String Slicing

Slicing means taking part of a string.

syntax 
string[start:end]

Example — start at index 0, stop before index 3:

name = "Nimal"
print(name[0:3])   # Nim
print(name[2:5])   # mal
Output
Nim
mal

Useful shortcut — omit the start to begin from the beginning:

print(name[:3])   # Nim — start from beginning, stop before 3
04

String Length

Use len() to count characters:

name = "Nimal"
print(len(name))
Output
5
Mistake 1 — Starting from 1

name[1] is not the first letter — it is the second. Index 0 is always the first character.

Mistake 2 — Invalid index

print(name[10]) gives an error because index 10 does not exist in a 5-character string.

05

What Is String Formatting?

String formatting means inserting variable values into text. Instead of printing separately:

name = "Nimal"
age = 20
print(name)
print(age)

We can combine them into one readable sentence:

Goal
My name is Nimal and I am 20 years old.

Python gives us four ways to do this.

06

Method 1 — Using Commas

method1.py 
name = "Nimal"
age = 20
print("My name is", name, "and I am", age, "years old.")
Output
My name is Nimal and I am 20 years old.
07

Method 2 — String Concatenation Using +

method2.py 
name = "Kamal"
print("Hello " + name)
Output
Hello Kamal
Numbers need str() first You cannot concatenate a string and a number directly. Convert the number first: 
age = 25
print("Age: " + str(age))
08

Method 3 — Old-Style Formatting Using %

This is an older formatting method. Placeholders inside the string are replaced by values.

PlaceholderMeaning
%dinteger
%ffloat
%sstring
method3.py 
age = 20
print("You are %d years old" % age)

name = "Sunil"
print("My name is %s" % name)

salary = 25000.50
print("Salary = %.2f" % salary)   # limit to 2 decimal places

name = "Kamal"
age = 30
print("%s is %d years old" % (name, age))
Output
You are 20 years old
My name is Sunil
Salary = 25000.50
Kamal is 30 years old
Why % formatting is less preferred now Although useful, it is an older style. Modern Python usually prefers f-strings.
09

Method 4 — f-strings (Recommended)

f-strings are the modern, recommended way to format strings. Prefix the string with f and place variable names inside {}.

method4.py 
name = "Nimal"
age = 20
print(f"My name is {name} and I am {age} years old.")
Output
My name is Nimal and I am 20 years old.

f-strings also support decimal formatting:

marks = 87.5678
print(f"Marks = {marks:.2f}")
Output
Marks = 87.57

You can even put expressions directly inside {}:

a = 10
b = 5
print(f"{a} + {b} = {a+b}")
Output
10 + 5 = 15
10

Escape Characters

Escape characters let you add special formatting inside strings.

escape.py 
name = "Nimal"

# \n — new line
print(f"Student Report\nName: {name}")

# \t — tab
print("Name\tAge")
print("Sunil\t25")
Output
Student Report
Name: Nimal

Name    Age
Sunil   25
11

Comments

A comment is a note for humans. Python ignores comments completely — they do not affect how the program runs.

Single-line comments — use #:

# storing student name
name = "Nimal"
print(name)

Inline comments — placed after code on the same line:

age = 20  # student's age
print(age)

Multi-line comments — use triple quotes:

"""
Student marks program
Author: Nimal Perera
Year: 2026
"""
12

String Literals Are Not Comments

Wrong — looks like a comment but isn't 
"Prints name"
print(name)
Correct 
# Prints name
print(name)

Python treats "Prints name" as a string, not a comment. Beginners should avoid this style and use # instead.

Proper use of string literals: docstrings Triple-quoted strings placed at the start of a function are called docstrings — a special case we will study later with functions: 
def greet():
    """Print greeting message"""
    print("Hello")
13

Real Example — Student Report

student_report.py 
# Student report program
name = input("Enter student name: ")
marks = float(input("Enter marks: "))

print(f"Student: {name}")
print(f"Marks: {marks:.2f}")
Example Output
Student: Nimal Silva
Marks: 88.50

Lesson Summary

  • Strings are made of characters; each has an index starting from 0. Use name[0], name[-1], name[0:3], and len(name) to access and measure them
  • Method 1 — commas: print("Hello", name)
  • Method 2 — concatenation: print("Hello " + name) — numbers need str()
  • Method 3 — old-style %: print("Age: %d" % age)
  • Method 4 (best) — f-strings: print(f"Age: {age}") — supports expressions and :.2f decimal formatting
  • Single-line comments use #; multi-line use triple quotes. Avoid bare string literals as fake comments
