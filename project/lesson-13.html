Lesson 13

Data Structures in Python

So far, your variables stored only one value at a time. Data structures let you store multiple values in a single variable — a fundamental need for any real program.

By the end of this lesson you will be able to

  • Store multiple values in one variable using data structures
  • Understand the differences between lists, tuples, sets, and dictionaries
  • Access, update, and remove data stored in Python data structures
  • Select the appropriate data structure for a given task
01

Why Do We Need Data Structures?

So far, we used variables like this:

name = "John"
age = 25

But what if you want to store multiple values? For example: 5 student names, 10 numbers, or a list of marks? You cannot create hundreds of separate variables. Instead, you use data structures.

Python has four main data structures for beginners List, Tuple, Set, and Dictionary — each with different properties for different purposes.
02

Lists — The Most Important Data Structure

A list stores multiple values in order, using square brackets.

numbers = [10, 20, 30, 40]
names = ["John", "Ann", "Mike"]

Accessing Values (Indexing) — Index starts from 0.

print(numbers[0])   # 10 (first item)
print(names[1])    # Ann (second item)
Programmers count from 0 This is one of the most common mistakes beginners make. The first item is at index 0, not 1. That's also why this course started with Lesson 0 instead of Lesson 1.

Changing Values

numbers[1] = 25
print(numbers)
Output
[10, 25, 30, 40]

Adding Items

numbers.append(50)

Removing Items

numbers.remove(30)
03

Tuples — Like Lists, But Fixed

A tuple is like a list, but it cannot be changed after it is created. Tuples use parentheses.

numbers = (10, 20, 30)
print(numbers[0])   # 10
Tuples are immutable — you cannot change their values
Wrong — TypeError 
numbers[1] = 50   # TypeError: tuple does not support item assignment
Use a list if you need to change values 
numbers = [10, 20, 30]
numbers[1] = 50   # Works

Use tuples when the data should not change — for example, days of the week or coordinates of a location.

04

Sets — No Duplicates

A set stores unique values only — duplicates are automatically removed. Sets use curly braces.

numbers = {1, 2, 3, 3, 4}
print(numbers)
Output
{1, 2, 3, 4}

The duplicate 3 is removed automatically.

Adding and Removing Items

numbers.add(5)
numbers.remove(2)
Sets are unordered You cannot access items in a set using an index. Sets have no guaranteed order. Use sets when you only care about unique values, not their position.
05

Dictionaries — Key-Value Pairs

A dictionary stores data as key-value pairs. Each piece of data has a unique name (key) that you use to access it.

student = {"name": "John", "age": 20, "grade": "A"}
print(student["name"])   # John
print(student["age"])    # 20

Changing Values

student["age"] = 21

Adding New Key-Value Pairs

student["city"] = "Colombo"

Removing a Key-Value Pair

del student["grade"]
06

Choosing the Right Data Structure

Data Structure Ordered? Changeable? Duplicates? Use When
ListYesYesYesOrdered, changeable collection
TupleYesNoYesFixed data that should not change
SetNoYesNoUnique items only, order doesn't matter
DictionaryYes (Python 3.7+)YesKeys: NoData needs labels (key-value pairs)

Lesson Summary

  • Lists ([]) store ordered, changeable values — the most commonly used data structure
  • List indexing starts at 0, not 1
  • Tuples (()) are like lists but immutable — use when data should not change
  • Sets ({}) store only unique values and are unordered — no index access
  • Dictionaries ({key: value}) store labeled data as key-value pairs
  • Choose the right data structure based on whether you need ordering, uniqueness, and mutability
