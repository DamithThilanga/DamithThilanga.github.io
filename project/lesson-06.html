Lesson 6

Naming Variables in Python: Writing Clean and Meaningful Code

At first glance, naming variables sounds like a small detail. In reality, it makes a huge difference in how easy your code is to read, understand, and maintain.

By the end of this lesson you will be able to

  • Apply Python variable naming rules correctly
  • Distinguish between valid and invalid variable names with reasons
  • Use meaningful variable names to improve code readability
  • Follow basic variable naming conventions in Python to write professional code
01

Why Is Variable Naming Important?

Imagine reading this:

x = 25
y = 10
z = x * y

Now compare it with this:

length = 25
width = 10
area = length * width

Both programs do the same thing. But the second one is much easier to understand. Good variable names make your code readable, help others understand your logic, reduce mistakes, and make debugging easier.

02

Basic Rules for Naming Variables

These are the mandatory rules — your program will not work properly if you break them.

Rule 1: Variable names can only contain letters (a–z, A–Z), numbers (0–9), and underscores (_). Other special characters are not allowed.

student_name = "John"    # valid
marks1 = 85              # valid

Rule 2: Variable names cannot start with a number.

Wrong 
1name = "John"    # SyntaxError
Correct 
name1 = "John"

Rule 3: Variable names cannot be Python keywords. Keywords are reserved words with special meanings in Python.

# These are invalid variable names:
# if, else, elif, while, for, def, return, True, False, None

Rule 4: Python variable names are case-sensitive. Name, name, and NAME are three completely different variables.

name = "John"
Name = "Mary"
# These are two different variables
03

Naming Conventions

Beyond the mandatory rules, Python has widely accepted conventions for naming variables. These are not enforced by Python itself, but following them makes your code professional and readable.

Use snake_case — separate words with underscores, all lowercase. This is the Python standard.

Python Convention 
student_name = "John"
total_marks = 450
is_passed = True
Avoid 
studentName = "John"  # camelCase
TotalMarks = 450   # PascalCase

Use meaningful names — the name should clearly describe what the variable stores.

meaningful.py 
# Bad — unclear purpose
a = 5
b = 3

# Good — purpose is immediately clear
number_of_students = 5
passing_marks = 3

Use boolean prefixes for boolean variables — prefix with is_, has_, or can_.

is_active = True
has_passed = False
can_vote = True
04

Common Beginner Mistakes

Using spaces in variable names
Wrong 
student name = "John"  # SyntaxError
Correct 
student_name = "John"
Using Python keywords as variable names
Wrong 
if = 10       # SyntaxError
print = 5    # overwrites built-in
Correct 
condition = 10
value = 5

Lesson Summary

  • Variable names can only contain letters, digits, and underscores — no spaces or special characters
  • Variable names cannot start with a number
  • Variable names cannot be Python keywords like if, else, or True
  • Python variable names are case-sensitive
  • Use snake_case (e.g. student_name) as the Python naming convention
  • Choose meaningful names that clearly describe what the variable stores
