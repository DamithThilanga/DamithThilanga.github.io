Lesson 2

Writing Useful Programs

"My program somehow works and gives a result" alone is not enough. Good programs have several essential qualities. Learning these fundamentals from the beginning will make you a better programmer — bad habits, once learned, are very difficult to unlearn.

By the end of this lesson you will be able to

  • Explain the characteristics of a useful computer program, comparing with real-world software
  • Recognize the importance of problem-solving before writing code
  • Apply basic good programming practices such as readability and proper organization from the very first codes you write
  • Understand the importance of testing and debugging programs
01

Usefulness

First, the program you write should be useful. If a task can be easily done with the existing method, you don't need to write a program for it in the first place. Writing a program to calculate someone's age from their birth year won't be of much use because most people can calculate that with their brains or a simple calculator.

Key question before you start Does this program make a task easier, faster, or more reliable than the current method? If the answer is no, reconsider whether the program is needed.
02

Accuracy and Reliability

The results of your program must always be accurate. A program written to calculate BMI (body mass index) that gives wrong answers most of the time is absolutely useless. If that program often crashes and users are not sure when it will work properly, that's another bad sign. Such a program is not reliable. Reliability is an essential property of a computer program.

Your computer program should also give results faster and more conveniently than the currently available technology. A BMI calculator that always gives correct results but takes one hour per calculation is useless — you can do it with pen and paper faster.

03

Exception Handling

An important concept in making good programs is exception handling. Think about an ATM machine. What happens if someone inserts an invalid card? Since the machine is not programmed to process that card, the program would crash, depriving other users until the machine is rebooted. But that doesn't happen in real life — the program is designed to handle the exception of an invalid card.

Applying the same concept to a BMI calculator: if the user enters 0 as the height, the program will crash trying to divide by zero. You should add code to handle that exception. Another user might enter their name instead of weight — that exception should also be handled.

We will learn this properly later Exception handling using try and except is covered in Lesson 19. For now, just understand why it matters.
04

User Friendliness

User friendliness is another important feature of a computer program. Simply put, it describes how easily your program allows users to use it. Remember the last time you visited an e-commerce website. How easy was it to navigate? Did you have to ask anyone how to place an order? Hopefully not — the interface was self-explanatory.

When designing user interfaces, it's better to comply with widely accepted norms. Fonts, colors, sizes, and even the positions of certain icons are significant. If you place the cart icon, search bar, and menu in unexpected positions, users will be confused and unlikely to engage with your program.

Commercial impact If your program is a commercial product like an e-commerce website, you should make it as attractive and convenient as possible. Poor user friendliness drives users to competitors.
05

Efficient Use of Resources

You should think about the efficient use of the computer's resources. Your algorithms should be written in the most efficient way, using the minimum amount of processing power, RAM, hard disk space, and other resources. If your program slows down the computer or demands closing other software, users will consider alternatives. This is one reason some users prefer Firefox over Chrome.

06

Readability and Collaboration

Most of the time, you are not the only person coding in a project. Therefore, others should be able to understand what you have done. Your code should be written taking that into consideration. This includes commenting, rational use of variable names, clear algorithms, and many other practices. If these are done poorly, even you might find it difficult to understand your own program later.

Coming up We will discuss inline commenting in detail in Lesson 16.

Lesson Summary

  • A good program must be useful — it should make tasks easier, faster, or more reliable
  • Accuracy and reliability are essential — programs must produce correct results consistently
  • Exception handling prevents crashes from unexpected user input
  • User-friendly programs follow accepted interface norms and are easy to navigate
  • Efficient programs make good use of CPU, memory, and disk resources
  • Readable code — with good comments and variable names — is easier to maintain and share
