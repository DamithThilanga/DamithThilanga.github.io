Lesson 19

Python Error Handling

Until now, we have written programs assuming everything goes correctly. But real users make mistakes. Programs should handle such mistakes properly — without crashing. This is where error handling becomes important.

By the end of this lesson you will be able to

  • Explain common runtime errors in Python with practical examples
  • Use try and except blocks to handle errors
  • Handle common exceptions such as ValueError and ZeroDivisionError for maximum reliability
  • Improve program reliability through exception handling
01

What Is an Error?

An error is a problem that causes a program to stop or behave unexpectedly. For example:

print(10 / 0)   # ZeroDivisionError — cannot divide by zero
number = int("hello")   # ValueError — "hello" is not an integer

Without error handling — the program crashes, the user gets a bad experience, and data may be lost.

With error handling — the program continues safely, the user gets a friendly message, and code becomes more robust.

Handling Errors with try and except

Python provides try and except to handle errors safely.

syntax 
try:
    # risky code
except:
    # code that runs if an error happens
1Python runs the code inside try
2If no error occurs, the program continues normally
3If an error occurs, Python jumps to except and runs that block instead

Example:

try_except.py 
try:
    number = int(input("Enter number: "))
    print(number)
except:
    print("Invalid input")
User enters: 10

Program does not crash. Much better.

04

Catching Specific Errors

Instead of catching everything, it is better to catch specific error types:

specific_except.py 
try:
    number = int(input("Enter number: "))
except ValueError:
    print("Please enter a valid integer")
Why be specific? Catching a named error makes your intent clear and avoids accidentally hiding unrelated bugs. A bare except: catches everything — including errors you didn't expect.
05

Multiple except Blocks

Different errors can be handled differently:

multi_except.py 
try:
    a = int(input("Enter number: "))
    result = 10 / a
    print(result)
except ValueError:
    print("Invalid number")
except ZeroDivisionError:
    print("Cannot divide by zero")

Using else

else runs only if no error occurred in the try block:

try_else.py 
try:
    num = int(input("Enter number: "))
except ValueError:
    print("Invalid input")
else:
    print("Valid input")
Input: 25
Valid input

09

Good Practices

Catch specific errors Use except ValueError: rather than a bare except: — specific errors are clearer and safer.
Bad practice — silently ignoring all errors
try:
    risky_code()
except:
    pass   # silently swallows every error — not recommended

This hides problems and makes debugging very difficult.

Give helpful messages
Bad 
print("Error")
Better 
print("Please enter a valid integer")
10

Common Beginner Mistakes

Mistake 1 — Forgetting indentation inside try
Wrong — IndentationError 
try:
print("Hello")
Correct 
try:
    print("Hello")
Mistake 2 — Using except without try

except must always follow a try block. Writing except: alone is a SyntaxError.

Lesson Summary

  • Common errors include SyntaxError, NameError, TypeError, ValueError, ZeroDivisionError, and IndexError
  • try / except lets you handle errors gracefully — the program continues instead of crashing
  • Catch specific error types (except ValueError:) rather than using a bare except:
  • Multiple except blocks can handle different error types differently
  • else runs only when no error occurs; finally always runs regardless
  • Never silently swallow errors with except: pass — always give users a helpful message
