Lesson 14

Repetitive Tasks with Loops

In programming, we often need to repeat a task multiple times. Writing the same code repeatedly is inefficient. Loops allow us to execute a block of code multiple times automatically.

By the end of this lesson you will be able to

  • Use for loops to iterate over sequences
  • Use while loops to repeat based on a condition
  • Use range() to generate number sequences
  • Control loop execution with break and continue
  • Combine loops with conditionals and data structures
01

Why We Need Loops

Without loops, printing numbers 1 to 5 looks like this:

print(1)
print(2)
print(3)
print(4)
print(5)

With a loop, the same result needs just two lines:

for i in range(1, 6):
    print(i)
02

The for Loop

The for loop runs once for each item in a sequence and stops when the items are over.

syntax 
for variable in sequence:
    # code block

Loop through a list:

numbers = [10, 20, 30]
for num in numbers:
    print(num)
Output
10
20
30

Loop through a string:

text = "Hello"
for ch in text:
    print(ch)

Each letter in the word is printed on a separate line.

03

Using range()

The range() function generates a sequence of numbers.

range(stop) — starts from 0, stops before the given number.

for i in range(5):
    print(i)
Output
0 1 2 3 4
range() does not include the stop value range(5) gives 0, 1, 2, 3, 4 — not 5. If you want to iterate from 5 to 10, write range(5, 11).

range(start, stop)

for i in range(1, 6):
    print(i)
Output
1 2 3 4 5

range(start, stop, step) — step decides how many numbers to skip.

for i in range(1, 10, 2):
    print(i)
Output
1 3 5 7 9
04

The while Loop

The while loop continues as long as its condition is True.

syntax 
while condition:
    # code block
example.py 
x = 1
while x <= 5:
    print(x)
    x = x + 1
Output
1 2 3 4 5
1 x starts as 1. Condition x <= 5 is True, so print x.
2 x becomes 2. Condition is still True. Print x.
3 This continues until x = 5, which is printed.
4 x becomes 6. Condition 6 <= 5 is False. Loop stops.
Infinite loop — forgetting to update the variable
Wrong — infinite loop! 
x = 1
while x <= 5:
    print(x)
# x is never updated — loop runs forever
Correct 
x = 1
while x <= 5:
    print(x)
    x = x + 1
05

Loop Control: break and continue

break — Stops the loop completely.

for i in range(1, 6):
    if i == 3:
        break
    print(i)
Output
1
2

Although the loop is set to run from 1 to 5, the break stops it when i reaches 3.

continue — Skips the current iteration and moves to the next one.

for i in range(1, 6):
    if i == 3:
        continue
    print(i)
Output
1
2
4
5

The number 3 is skipped but the loop continues for all other values.

06

Loops with Data Structures

Looping through a list:

marks = [60, 70, 80]
for mark in marks:
    print(mark)

Looping through a dictionary:

student = {"name": "John", "age": 20}
for key in student:
    print(key, student[key])
Output
name John
age 20
07

Combining Loops and Conditionals

even_odd.py 
numbers = [10, 15, 20, 25]
for num in numbers:
    if num % 2 == 0:
        print(num, "is even")
    else:
        print(num, "is odd")
Output
10 is even
15 is odd
20 is even
25 is odd
08

Common Beginner Mistakes

  • Forgetting to update variables in while loops (causes infinite loops)
  • Incorrect indentation inside loops
  • Confusing range() stop value — it is exclusive (not included)
  • Trying to index sets (not allowed — sets are unordered)
  • Modifying a list incorrectly inside a loop

Lesson Summary

  • Loops allow repeating code efficiently instead of writing the same lines multiple times
  • for loops iterate over sequences (lists, strings, range())
  • while loops run as long as a condition remains True
  • break stops a loop completely; continue skips the current iteration
  • range(start, stop, step) generates number sequences — stop value is not included
  • Loops can be combined with conditionals and data structures for powerful programs
