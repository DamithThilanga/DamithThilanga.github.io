Lesson 18

Python Built-in Functions, Modules, and Scope

So far we have learned how to write our own functions, use loops and conditionals, and work with strings and data structures. Now we learn three topics that make Python much more powerful: built-in functions, importing modules, and global vs local scope.

By the end of this lesson you will be able to

  • Use Python's common built-in functions to reduce and simplify code
  • Import and use modules such as math, random, and datetime
  • Explain the difference between local and global scope
  • Use the global keyword correctly and with care
01

Built-in Functions

Python comes with many ready-made functions that we can use directly — we do not need to write them ourselves. These are called built-in functions. You have already used one: print().

02

Common Built-in Functions

len() — find the length of a string or collection:

name = "Nimal"
print(len(name))
Output
5

type() — show the data type of a value:

x = 10
print(type(x))
Output
<class 'int'>

max() and min() — find largest and smallest values:

numbers = [5, 2, 9, 1]
print(max(numbers))
print(min(numbers))
Output
9
1

sum() — add all numbers in a list:

numbers = [10, 20, 30]
print(sum(numbers))
Output
60

round() — round a number to a given number of decimal places:

print(round(3.678, 2))
Output
3.68
Why built-in functions matter They save time, reduce code, improve readability, and avoid writing repeated logic.
03

Importing Modules

A module is a file containing Python code — functions, variables, and more — that we can reuse. Not everything is built into the core language; modules extend Python's functionality. We import them using the import keyword.

math module — mathematical operations:

math_example.py 
import math

print(math.sqrt(16))
Output
4.0

random module — generate random values:

random_example.py 
import random

print(random.randint(1, 10))   # random integer between 1 and 10

datetime module — work with dates and time:

datetime_example.py 
import datetime

print(datetime.datetime.now())

Importing specific functions — instead of importing the whole module:

from math import sqrt

print(sqrt(25))
Output
5.0
04

Scope — Local vs Global Variables

Scope means where a variable can be accessed in a program. There are two types: local and global.

Local scope — a variable created inside a function can only be used inside that function:

scope.py 
def test():
    x = 10
    print(x)    # Works fine

test()
print(x)        # NameError — x does not exist here

Global scope — a variable created outside a function can be used anywhere:

x = 20

def test():
    print(x)   # Works — x is global

test()
print(x)
Output
20
20
05

Local vs Global — When Names Clash

When the same name is used inside and outside a function, they are treated as separate variables:

scope_clash.py 
x = 5      # global x

def change():
    x = 10   # local x — separate from global
    print("Inside function:", x)

change()
print("Outside function:", x)
Output
Inside function: 10
Outside function: 5

The global x is not changed. The function created its own local x.

06

Using the global Keyword

If we want to modify a global variable inside a function, we must declare it with global:

global_keyword.py 
x = 5

def change():
    global x
    x = 10

change()
print(x)
Output
10
Use global with care Using global is allowed, but should be used sparingly. In most cases it is better to use function parameters and return values — this keeps code cleaner and easier to debug.

Lesson Summary

  • Built-in functions like len(), type(), max(), min(), sum(), and round() are ready to use without importing anything
  • Modules extend Python — import them with import math, import random, etc. Use from module import function to import specific items
  • A local variable lives inside its function only — it cannot be accessed outside
  • A global variable is created outside all functions and is accessible everywhere
  • When the same name exists inside and outside a function, they are separate variables — the function's local copy does not affect the global one unless you use global
