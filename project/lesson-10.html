Task 1: Name and City Message

Try it yourself before reading the solution.

name = input("What's your name? ")
city = input("Which city are you from? ")

02

Task 2: Adding Two Numbers — The Common Mistake

Let's write a program to add two numbers entered by the user and display the sum.

buggy.py 
number1 = input("Enter the first number: ")
number2 = input("Enter the second number: ")
sum = number1 + number2
print("The sum is:", sum)

Run this, enter two numbers, and see the result. The output is wrong — instead of adding the numbers, Python joined the strings. This is because input() always returns a string, and + on strings means concatenation.

String concatenation vs number addition If the user enters 2 and 3, the result will be 23 (two strings joined), not 5 (the sum).

Here is the corrected version:

fixed.py 
number1 = input("Enter the first number: ")
number2 = input("Enter the second number: ")
number1 = int(number1)
number2 = int(number2)
sum = number1 + number2
print("The sum is", sum)

Lines 3 and 4 do the type conversion from string to integer. Alternatively, you can do it all on one line:

number1 = int(input("Enter the first number: "))
number2 = int(input("Enter the second number: "))
print("The sum is", number1 + number2)

Task 3: Celsius to Kelvin

Write a program to convert temperature from Celsius to Kelvin. (To convert to Kelvin, add 273 to the Celsius value.)

temperature.py 
temperature_c = float(input("Enter the temperature in Celsius: "))
temperature_k = temperature_c + 273.0
print("Temperature in Kelvin:", temperature_k)
Why float() instead of int()? Temperatures can have decimal points (e.g. 36.5°C), so we use float() instead of int().

If we don't need to use temperature_k again, we can shorten the program to two lines:

temperature_c = float(input("Enter the temperature in Celsius: "))
print("Temperature in Kelvin:", temperature_c + 273.0)
In programming, there can be more than one correct answer The longer version and the shorter version are both correct. Choose whichever is clearer for the situation.

Task 4: Age Next Year

Write a program that gets the user's age and prints their age next year.

age.py 
age = int(input("How old are you now? "))
print("You will be", age + 1, "years old next year")
What if the user enters a word like "Hello"? The program will crash because int("Hello") raises an error. A good program should handle this — we will learn exception handling in Lesson 19. Until then, please don't enter strings when a number is expected.

Task 5: Average Marks of Five Students

Write a program to calculate the average marks of five students.

average.py 
marks1 = int(input("Enter marks of student 1: "))
marks2 = int(input("Enter marks of student 2: "))
marks3 = int(input("Enter marks of student 3: "))
marks4 = int(input("Enter marks of student 4: "))
marks5 = int(input("Enter marks of student 5: "))
average = (marks1 + marks2 + marks3 + marks4 + marks5) / 5
print("Average marks:", average)
A more efficient approach exists There is a more efficient way of doing this with a loop. You will learn about loops in Lesson 14 and can revisit this problem then.

Project: Simple Calculator

Build a basic calculator that gets two numbers from the user and performs addition, subtraction, multiplication, and division, displaying the result of each operation.

Try it yourself first, then compare your solution:

calculator.py 
num1 = int(input("Enter first number: "))
num2 = int(input("Enter second number: "))
print("Addition:", num1 + num2)
print("Subtraction:", num1 - num2)
print("Multiplication:", num1 * num2)
print("Division:", num1 / num2)
Example run
Enter first number: 10
Enter second number: 3
Addition: 13
Subtraction: 7
Multiplication: 30
Division: 3.3333333333333335

Lesson Summary

  • input() always returns a string — convert to int() or float() when doing arithmetic
  • You can do type conversion in the same line as input() to keep code shorter
  • String concatenation with + is different from number addition — this is a very common mistake
  • Use float() when the input may have a decimal point
  • There is often more than one correct solution to a programming problem
