Lesson 17

Python Functions

Until now, we have written Python programs line by line. Functions help us organize code into reusable blocks — like a small machine that takes input, does a task, and may give back a result.

By the end of this lesson you will be able to

  • Define functions using the def keyword
  • Call functions correctly
  • Pass arguments into functions
  • Return values using the return keyword
  • Organize code into reusable blocks to make more organised code
01

Why Do We Need Functions?

Consider a small program written line by line:

name = input("Enter your name: ")
print("Hello", name)

age = int(input("Enter your age: "))
print("You are", age, "years old")

This works well for small programs. But imagine writing a larger program with:

  • student registration
  • mark calculation
  • report generation
  • grade assignment

If we keep writing everything line by line, the code becomes:

  • long
  • repetitive
  • difficult to read
  • difficult to debug

This is where functions become useful. Think of a function as a small machine:

  • you give it something (input)
  • it does a task
  • it may give back a result (output)
02

What Is a Function?

A function is a named block of code designed to perform a specific task.

Instead of writing the same thing repeatedly:

print("Hello")
print("Hello")
print("Hello")

We can create a function and use it multiple times:

def greet():
    print("Hello")

Breaking it down:

1def means define a function
2greet is the function name
3() holds inputs — currently none
4: starts the function block
5The indented code belongs to the function
Definition vs execution Writing def greet(): only defines the function. It does not run yet. You must call it separately.
03

Calling a Function

To run a function, write its name followed by parentheses:

example.py 
def greet():
    print("Hello")

greet()
Output
Hello

You can call the same function multiple times:

greet()
greet()
greet()
Output
Hello
Hello
Hello

This is better than rewriting the same code three times.

04

Why Use Functions?

Functions help us in three important ways:

1. Reuse code — Instead of repeating code many times.

Bad 
print("Welcome")
print("Welcome")
print("Welcome")
Better 
def welcome():
    print("Welcome")

welcome()
welcome()
welcome()

2. Make code organised

Without functions 
name = input("Enter name: ")
print("Hello", name)

marks = int(input("Enter marks: "))
print("Marks:", marks)
With functions 
def greet_user():
    name = input("Enter name: ")
    print("Hello", name)

greet_user()

3. Easier debugging — If something breaks, you can inspect one function. Instead of checking 200 lines, maybe only 5 lines.

05

Function Syntax

General syntax:

syntax 
def function_name():
    statements

Example:

example.py 
def show_message():
    print("Python is fun")

show_message()
Output
Python is fun
06

Functions with Parameters

Sometimes functions need input. Instead of a fixed greeting:

def greet():
    print("Hello")

We may want a personalised greeting. Add a parameter:

example.py 
def greet(name):
    print("Hello", name)

greet("Nimal")
Output
Hello Nimal

The same function works with different inputs:

greet("Sunil")
greet("Kamal")
greet("John")
Output
Hello Sunil
Hello Kamal
Hello John
07

Multiple Parameters

Functions can take multiple inputs:

example.py 
def introduce(name, age):
    print(f"My name is {name} and I am {age} years old")

introduce("Nimal", 20)
Output
My name is Nimal and I am 20 years old
Important rule: order matters introduce("Nimal", 20) is correct. But introduce(20, "Nimal") gives the wrong meaning — Python produces no error, but the output becomes: My name is 20 and I am Nimal years old.
08

Return Values

Some functions produce results. Without return, you can only print:

def add():
    result = 5 + 3
    print(result)

But sometimes we want to store the result and use it elsewhere. Use return:

example.py 
def add():
    return 5 + 3

answer = add()
print(answer)
Output
8

Why return is more flexible:

def square():
    return 4 * 4

result = square()
print(result)          # 16

x = square() + 10
print(x)              # 26
09

Functions with Parameters and Return

This combination is the most common pattern in real programs:

example.py 
def add_numbers(a, b):
    return a + b

result = add_numbers(10, 5)
print(result)
Output
15

You can also pass the call directly to print():

print(add_numbers(4, 6))
print(add_numbers(100, 50))
Output
10
150
10

Local Variables

Variables created inside a function belong only to that function. This is called local scope.

scope.py 
def test():
    x = 10
    print(x)    # Works — x is visible here

test()

print(x)        # Error — x does not exist outside the function
Local scope Because x is defined inside test(), it exists only while the function is running. Once the function ends, x is gone.
11

The Difference Between print() and return

This confuses many beginners. Here is the clear distinction:

print() — displays value 
def test():
    print(5)

test()
return — sends value back 
def test():
    return 5

x = test()
print(x)

Watch what happens when you try to store the result of a print()-only function:

example.py 
def test():
    print(5)

x = test()
print(x)
Output
5
None
Why None? The function printed 5 but returned nothing. Python automatically stores None when a function has no return statement.
12

None and Functions Without Return

If a function has no return, Python automatically returns None. These two are equivalent:

What you write 
def hello():
    print("Hello")
What Python sees 
def hello():
    print("Hello")
    return None
13

Good Function Naming

Function names should clearly describe what the function does.

Bad — unclear 
def x():
    print("Hello")
Better — clear purpose 
def print_greeting():
    print("Hello")

Use descriptive names:

  • calculate_total()
  • print_report()
  • get_grade()
  • find_average()

Avoid vague names like abc(), temp(), or data1() unless truly temporary.

Comments in functions Add a short comment inside complex functions to describe what they do. This improves readability for you and others who read your code later. 
def add_numbers(a, b):
    # returns sum of two numbers
    return a + b
14

Complete Example — Student Average

average.py 
def calculate_average(mark1, mark2, mark3):
    average = (mark1 + mark2 + mark3) / 3
    return average

result = calculate_average(80, 90, 100)
print("Average =", result)
Output
Average = 90.0
15

Complete Example — Grade Calculator

grade_calc.py 
def get_grade(mark):
    if mark >= 75:
        return "A"
    elif mark >= 65:
        return "B"
    elif mark >= 50:
        return "C"
    else:
        return "F"

grade = get_grade(82)
print("Grade:", grade)
Output
Grade: A
16

Real Example — Student Report System

Functions can call each other to build larger programs:

report.py 
def get_total(mark1, mark2):
    return mark1 + mark2

def get_average(total):
    return total / 2

total = get_total(80, 90)
average = get_average(total)

print("Total =", total)
print("Average =", average)
Output
Total = 170
Average = 85.0
17

Common Beginner Mistakes

Mistake 1 — Forgetting parentheses when calling
Wrong 
greet   # does not call the function
Correct 
greet()  # parentheses are required
Mistake 2 — Forgetting indentation
Wrong — IndentationError 
def greet():
print("Hello")
Correct 
def greet():
    print("Hello")
Mistake 3 — Confusing print() and return

A function that uses print() only displays a value — it cannot be stored or used in calculations. A function with return sends the value back to the caller so it can be stored, printed, or used in further computation. Understand their difference clearly.

Lesson Summary

  • A function is a named block of code that performs a specific task — defined with def and called with functionname()
  • Parameters let you pass input into functions; multiple parameters are separated by commas and order matters
  • return sends a value back to the caller — more flexible than print() alone
  • Variables created inside a function are local — they exist only within that function (local scope)
  • A function with no return statement automatically returns None
  • Use descriptive function names like calculate_total() instead of x() or temp()
